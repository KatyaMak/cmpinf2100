pd.merge — Joining on a Key Column

Unlike concat (tape it), merge matches rows by value. Both DataFrames need at least one column with matching values — that's the key. What you keep depends on the join type.

The two DataFrames we'll join

Key column: street — exists in both, same dtype. Pine is only on the left. Maple is only on the right.

Inner Join — default

Keys in both → Elm, Oak

         how='right')

MapleNaN2.1

Outer Join

Everything → Elm, Oak, Pine, Maple

pd.merge(sq, si, on='street',
         how='outer')
streetconditionlength_km
ElmGood1.2
OakFair0.8
PinePoorNaN
MapleNaN2.1

Two things both need to be true

✓ matching values — "Elm" appears in both DataFrames, so it can be joined

✓ matching dtypes — both key columns need same type (both str, or both int)

One can fail without the other:
• Same dtype, no matching values → merge runs but returns empty DataFrame
• Matching values but different dtypes → merge may silently miss matches

melt() — Wide → Long Reshape

Wide: one column per variable, one row per observation.
Long: two columns — variable (name) and value — one row per measurement.
Seaborn and many stats functions want long format.

0Adeliemass_g3750
1Adeliebill_mm39.5
1Adelieflipper_mm186

penguins_long = penguins.melt(
    id_vars=['penguin_id', 'species']   # columns that stay as identifiers
)
# bill_mm, flipper_mm, mass_g → stacked into 'variable' + 'value'

The decision: what are id_vars?

id_vars = columns that identify the thing being measured — they should not get stacked.

Everything else becomes a row: each original column name goes into variable, the cell value goes into value.

Penguin 0 was one row → becomes 3 rows (one per measurement). 2 penguins × 3 measurements = 6 rows.
Why long? Seaborn's hue= parameter expects one column for color. In wide format, measurements are in separate columns and can't be hued together. In long format, all measurements are in one value column and the variable column tells seaborn what to color.

numeric_only vs select_dtypes — two paths, same destination

df.mean(numeric_only=True)

One-time instruction to that method. Ignore strings, just run.

Best for: one quick stat

vs
df.select_dtypes('number').mean()

Filters the DataFrame first → gives you a reusable numeric-only DataFrame.

Best for: using numeric cols repeatedly

map() vs apply()

.map() — one cell in, one cell out. Best for transforming a single column by a dictionary or simple function.

.apply() — sees an entire row (axis=1) or column at once. Use when the result depends on multiple values.

Missing values — dropna()

df.dropna()                    # drop any row with ANY NaN
df.dropna(how='all')           # drop row only if ALL values are NaN
df.dropna(subset=['race_2'])   # drop only if this specific column is NaN
df.dropna(axis=1)              # drop columns instead of rows
Remember: dropna() is a method on the DataFrame — it's df.dropna(), not pd.dropna(df).

size vs count — don't mix them up

.size() — rows per group, including NaN
.count() — non-NaN values per group

If your data has missing values, these will differ. count silently skips NaN rows — use size if you want the true group total.

.agg() — multiple functions at once

# Simple: same functions on multiple columns
df.groupby("E")[['B', 'C']].agg(['mean', 'std'])

# Named aggregation: one output column per line
df.groupby("E").agg(
    B_avg  = ('B', 'mean'),   # output_name = (source_col, action)
    B_spread = ('B', 'std'),
    C_total  = ('C', 'sum')
)
The rule inside .agg(): output_name = (source_column, action)

Always a tuple, always two things: which column and what to do. The output name is whatever you want.

String shortcuts inside agg

StringWhat it does

Multi-column groupby

df.groupby(["E", "F"], dropna=False)["B"].mean().reset_index()
# groups by E AND F together → subgroup summaries
# reset_index() turns group keys back into regular columns (not index)
reset_index() after groupby: group keys become the index after aggregation. They look fine when printed, but break when you try to plot, filter, or merge on them. reset_index() rescues them into normal columns.
